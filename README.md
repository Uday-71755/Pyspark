🔍 Project Overview
The goal of this project was to design and implement a robust data processing workflow—from data ingestion to transformation and storage—leveraging the distributed computing power of Apache Spark.

⚙️ Key Concepts & Skills Applied
*Built a strong foundation by understanding Apache Spark architecture, including jobs, stages, and tasks.
*Implemented lazy evaluation and optimized execution using Spark DAGs.
*Performed data ingestion from CSV and JSON sources using Spark Data Reader APIs.
*Designed structured schemas using StructType and DDL schema definitions.
*Applied data transformations at beginner, intermediate, and advanced levels.
*Used window functions for advanced analytics (ranking, aggregation, partitioning).
*Created and optimized User Defined Functions (UDFs).
*Managed data writing with different modes (overwrite, append, etc.).
*Worked with efficient storage formats like Parquet.
*Explored Managed vs External tables in Spark.
*Queried data using Spark SQL for analytical insights.

📊 Outcome
*Built scalable ETL pipelines capable of handling structured and semi-structured data.
*Improved query performance using optimized transformations and storage formats.
*Gained practical experience working in a cloud-based environment with Databricks.
